2. Executive Summary & Architecture Overview

The Intervention Management Platform serves as the mission-critical logging, routing, and audit ledger tracking framework for pipeline incidents, contamination boundaries, and infrastructure failures inside the regulated United Kingdom water sector. Given the high compliance tracking parameters required by sector regulators, authentication boundaries must maintain absolute precision, multi-role security parameters, and verifiable access histories.

This document details the complete end-to-end transformation of the system security boundary. Moving away from a legacy local SQL authentication engine, the platform has integrated an enterprise ecosystem led by Microsoft Entra ID Single Sign-On (SSO). The new design preserves local developer capabilities and emergency support access by supporting a parallel dual-login framework driven by infrastructure feature flags.

The overarching architecture addresses three significant distributed authentication traps:

  1. The Timing Race Condition: Shifting the Microsoft Graph security group lookup out of deferred request handlers and nesting it directly into the OpenID Connect server-side token validation lifecycle event block, pre-allocating identity contexts.
  2. The Data Boundary Mismatch: Handling structural typos generated by corporate directory creation teams (such as -REQUESTOR with an 'O' vs database schema -REQUESTER with an 'E') immediately at the front gate entry point via inline normalization code blocks.
  3. The Transitional Identity Stale Cache: Implementing a fallback-aware dynamic memory caching lifespan configuration inside the UI engine to prevent initial provisioning loops from locking a user account out of their designated administrative dashboard layout configurations.

3. Database Architecture (RBAC Schema)

The system maintains resource separation boundaries utilizing a traditional normalized relational Role-Based Access Control configuration residing inside the [IMT] SQL Server schema layout directory. The data model uses strict foreign key relationships to prevent orphan data entries across security bounds.

3.1 Data Dictionary Specifications

3.2 Business Priority Single-Role Matrix Hierarchy

When an enterprise corporate account is associated with multiple directory security groups simultaneously, the application front-end simplifies the claims array into exactly **one active operational identity context**. Ranks are assigned lower values based on organizational weight:

Priority Hierarchy Formulation: ADMIN (Weight 1) → APPROVER (Weight 2) → SITEMANAGER (Weight 3) → REQUESTER (Weight 4) → DEVELOPER (Weight 5) → UNKNOWN (Weight 99)

This prioritization is enforced during serialization routines using specialized LINQ query filters, ensuring predictable UI layouts and clear component visibility across the platform.

4. UI Application Architecture (Frontend MVC Gateway)

The MVC frontend application serves as the primary authentication gateway, managing OpenID Connect challenge workflows, handling cookie sessions, and passing security contexts down to backend resource APIs.

4.1 `AuthenticationServiceExtensions.cs`

This code establishes the dual authentication pipeline. It securely chains onto Microsoft Identity's token validation events to pre-fetch group memberships from the Microsoft Graph API, while smoothly fallback-routing to standard cookies if SSO is disabled.

using System;
using System.Collections.Generic;
using System.Net.Http;
using System.Net.Http.Headers;
using System.Text.Json;
using System.Threading.Tasks;
using Microsoft.AspNetCore.Authentication.Cookies;
using Microsoft.AspNetCore.Authentication.OpenIdConnect;
using Microsoft.AspNetCore.Hosting;
using Microsoft.Extensions.Caching.Memory;
using Microsoft.Extensions.Configuration;
using Microsoft.Extensions.DependencyInjection;
using Microsoft.Identity.Web;
using NLog;

namespace InterventionsManagementUI.AppFrontend.Extensions
{
    public static class AuthenticationServiceExtensions
    {
        private static readonly Logger _logger = LogManager.GetCurrentClassLogger();

        public static IServiceCollection AddDualAuthentication(this IServiceCollection services, IConfiguration configuration)
        {
            var enableSso = configuration.GetValue<bool>("AuthenticationFeatures:EnableSSO");
            var jwtSettings = configuration.GetSection("JwtSettings");
            
            var authBuilder = services.AddAuthentication(options =>
            {
                options.DefaultScheme = CookieAuthenticationDefaults.AuthenticationScheme;
                options.DefaultChallengeScheme = OpenIdConnectDefaults.AuthenticationScheme;
            });

            authBuilder.AddCookie(CookieAuthenticationDefaults.AuthenticationScheme, options =>
            {
                options.Cookie.Name = "InterventionsAuthCookie";
                options.Cookie.HttpOnly = true;
                options.Cookie.Path = "/";
                options.Cookie.SameSite = Microsoft.AspNetCore.Http.SameSiteMode.Lax;
                options.LoginPath = "/Account/Login";
            });

            if (enableSso)
            {
                var apiScopes = configuration.GetSection("DownstreamApi:Scopes").Get<string[]>() ?? Array.Empty<string>();

                authBuilder
                    .AddMicrosoftIdentityWebApp(configuration.GetSection("AzureAd"))
                    .EnableTokenAcquisitionToCallDownstreamApi(apiScopes)
                    .AddInMemoryTokenCaches();

                services.Configure<OpenIdConnectOptions>(OpenIdConnectDefaults.AuthenticationScheme, options =>
                {
                    options.SaveTokens = true;
                    var existingOnTokenValidatedHandler = options.Events.OnTokenValidated;

                    options.Events.OnTokenValidated = async context =>
                    {
                        if (existingOnTokenValidatedHandler != null)
                        {
                            await existingOnTokenValidatedHandler(context);
                        }

                        var tokenAcquisition = context.HttpContext.RequestServices.GetRequiredService<ITokenAcquisition>();
                        var cache = context.HttpContext.RequestServices.GetRequiredService<IMemoryCache>();
                        var env = context.HttpContext.RequestServices.GetRequiredService<IWebHostEnvironment>();

                        var oid = context.Principal?.FindFirst("oid")?.Value 
                               ?? context.Principal?.FindFirst("http://schemas.microsoft.com/identity/claims/objectidentifier")?.Value;

                        if (!string.IsNullOrEmpty(oid))
                        {
                            string cacheKey = $"UI_AppRoles_{oid.ToLowerInvariant()}";
                            _logger.Info($"[SSO HANDSHAKE] Intercepted OIDC Validation Event. Warming cache key: '{cacheKey}'");

                            try
                            {
                                var graphToken = await tokenAcquisition.GetAccessTokenForUserAsync(
                                    scopes: new[] { "User.Read" },
                                    authenticationScheme: OpenIdConnectDefaults.AuthenticationScheme,
                                    user: context.Principal
                                );

                                using var client = new HttpClient();
                                client.DefaultRequestHeaders.Authorization = new AuthenticationHeaderValue("Bearer", graphToken);
                                var response = await client.GetAsync("https://graph.microsoft.com/v1.0/me/appRoleAssignments");
                                
                                if (response.IsSuccessStatusCode)
                                {
                                    var json = await response.Content.ReadAsStringAsync();
                                    using var doc = JsonDocument.Parse(json);
                                    var extractedGroups = new HashSet<string>();
                                    
                                    string envShortcode = env.EnvironmentName.ToUpper() switch 
                                    { 
                                        "LOCAL" => "DEV", "DEV" => "DEV", "UAT" => "UAT", "PROD" => "PROD", _ => "DEV" 
                                    };

                                    if (doc.RootElement.TryGetProperty("value", out var rolesArray))
                                    {
                                        foreach (var role in rolesArray.EnumerateArray())
                                        {
                                            if (role.TryGetProperty("principalDisplayName", out var nameProp))
                                            {
                                                var groupName = nameProp.GetString()?.ToUpper();
                                                if (!string.IsNullOrEmpty(groupName) && 
                                                    groupName.Contains("SG-APPC-INTERVENTIONS") && 
                                                    groupName.Contains($"-{envShortcode}-"))
                                                {
                                                    // CRITICAL INTERACTION FIX: Normalize IT typo suffix variation safely
                                                    if (groupName.Contains("-REQUESTOR"))
                                                    {
                                                        groupName = groupName.Replace("-REQUESTOR", "-REQUESTER");
                                                    }
                                                    extractedGroups.Add(groupName);
                                                }
                                            }
                                        }
                                    }

                                    string appRolesHeaderValue = string.Join(",", extractedGroups);
                                    cache.Set(cacheKey, appRolesHeaderValue, TimeSpan.FromMinutes(60));
                                    _logger.Info($"[SSO HANDSHAKE] Cache successfully primed for OID '{oid}' with groups: '{appRolesHeaderValue}'");
                                }
                            }
                            catch (Exception ex)
                            {
                                _logger.Error(ex, $"[SSO HANDSHAKE CRITICAL ERROR] Graph API collection execution mapping failed: {ex.Message}");
                            }
                        }
                    };
                });
            }
            else
            {
                // Graceful registration fallback to prevent DI container instantiating crashes under local dev builds
                services.AddScoped<ITokenAcquisition>(provider => null!);
                _logger.Warn("[AUTH CONFIG TRACE] SSO is explicitly disabled. Null factory registered for ITokenAcquisition constraints.");
            }

            return services;
        }
    }
}

4.2 `AuthTokenHandler.cs`

An outbound HTTP message handler that intercepts calls heading to downstream resource services. It injects the active Bearer token and appends custom app role claims to ensure clean security handshakes.

using System;
using System.Net.Http;
using System.Net.Http.Headers;
using System.Threading;
using System.Threading.Tasks;
using Microsoft.AspNetCore.Authentication.OpenIdConnect;
using Microsoft.AspNetCore.Http;
using Microsoft.Extensions.Caching.Memory;
using Microsoft.Extensions.Configuration;
using Microsoft.Identity.Web;
using NLog;

namespace InterventionsManagementUI.Infrastructure.HTTPHandler
{
    public class AuthTokenHandler : DelegatingHandler
    {
        private static readonly Logger _logger = LogManager.GetCurrentClassLogger();
        private readonly IHttpContextAccessor _httpContextAccessor;
        private readonly IConfiguration _configuration;
        private readonly IMemoryCache _cache;
        private readonly ITokenAcquisition _tokenAcquisition;

        public AuthTokenHandler(IHttpContextAccessor httpContextAccessor, IConfiguration configuration, IMemoryCache cache, ITokenAcquisition tokenAcquisition)
        {
            _httpContextAccessor = httpContextAccessor;
            _configuration = configuration;
            _cache = cache;
            _tokenAcquisition = tokenAcquisition;
        }

        protected override async Task<HttpResponseMessage> SendAsync(HttpRequestMessage request, CancellationToken cancellationToken)
        {
            var httpContext = _httpContextAccessor.HttpContext;
            var requestPath = request.RequestUri?.ToString() ?? "Unknown";

            if (httpContext?.User?.Identity != null && httpContext.User.Identity.IsAuthenticated)
            {
                var isSsoUser = httpContext.User.HasClaim(c => c.Type == "preferred_username" || c.Type == "http://schemas.microsoft.com/identity/claims/objectidentifier");

                if (isSsoUser && _tokenAcquisition != null)
                {
                    try
                    {
                        var apiScopes = _configuration.GetSection("DownstreamApi:Scopes").Get<string[]>() ?? Array.Empty<string>();
                        var apiToken = await _tokenAcquisition.GetAccessTokenForUserAsync(
                            scopes: apiScopes,
                            authenticationScheme: OpenIdConnectDefaults.AuthenticationScheme,
                            user: httpContext.User
                        );

                        if (!string.IsNullOrEmpty(apiToken))
                        {
                            request.Headers.Authorization = new AuthenticationHeaderValue("Bearer", apiToken);
                        }

                        var oidClaim = httpContext.User.FindFirst("http://schemas.microsoft.com/identity/claims/objectidentifier") ?? httpContext.User.FindFirst("oid");
                        if (oidClaim != null)
                        {
                            string cacheKey = $"UI_AppRoles_{oidClaim.Value.ToLowerInvariant()}";

                            if (_cache.TryGetValue(cacheKey, out string appRolesHeaderValue) && !string.IsNullOrEmpty(appRolesHeaderValue))
                            {
                                if (request.Headers.Contains("X-Entra-App-Roles"))
                                {
                                    request.Headers.Remove("X-Entra-App-Roles");
                                }
                                request.Headers.Add("X-Entra-App-Roles", appRolesHeaderValue);
                                _logger.Info($"[HANDLER TRACE] Stamped pre-fetched roles header for path '{requestPath}' directly from cache window.");
                            }
                        }
                    }
                    catch (Exception ex)
                    {
                        _logger.Error(ex, $"[HANDLER CRITICAL FAILURE] Outbound bearer attachment generation aborted: {ex.Message}");
                    }
                }
            }

            return await base.SendAsync(request, cancellationToken);
        }
    }
}

4.3 `UiRbacClaimsTransformer.cs`

Manages user session parsing in the UI. It uses a direct header injection strategy to bypass initial execution race conditions, and applies a short-lived cache lifespan to protect fallback profiles from data-stale locks.

using System;
using System.Collections.Generic;
using System.Linq;
using System.Net.Http;
using System.Net.Http.Headers;
using System.Security.Claims;
using System.Text.Json;
using System.Threading.Tasks;
using Microsoft.AspNetCore.Authentication;
using Microsoft.AspNetCore.Authentication.OpenIdConnect;
using Microsoft.Extensions.Caching.Memory;
using Microsoft.Extensions.Configuration;
using Microsoft.Identity.Web;
using NLog;

namespace InterventionsManagementUI.AppFrontend.Extensions
{
    public class UiRbacClaimsTransformer : IClaimsTransformation
    {
        private static readonly Logger _logger = LogManager.GetCurrentClassLogger();
        private readonly IMemoryCache _cache;
        private readonly IConfiguration _configuration;
        private readonly IHttpClientFactory _httpClientFactory;
        private readonly ITokenAcquisition _tokenAcquisition;

        public UiRbacClaimsTransformer(IMemoryCache cache, IConfiguration configuration, IHttpClientFactory httpClientFactory, ITokenAcquisition tokenAcquisition)
        {
            _cache = cache;
            _configuration = configuration;
            _httpClientFactory = httpClientFactory;
            _tokenAcquisition = tokenAcquisition;
        }

        public async Task<ClaimsPrincipal> TransformAsync(ClaimsPrincipal principal)
        {
            if (principal?.Identity == null || !principal.Identity.IsAuthenticated) return principal;
            if (principal.HasClaim(c => c.Type == "UiRbacTransformed")) return principal;

            var oidClaim = principal.FindFirst("http://schemas.microsoft.com/identity/claims/objectidentifier") ?? principal.FindFirst("oid");
            if (oidClaim == null) return principal;

            string azureAdObjectId = oidClaim.Value;
            string cacheKey = $"UiUserClaims_{azureAdObjectId.ToLowerInvariant()}";

            if (!_cache.TryGetValue(cacheKey, out UserProfileDto profile))
            {
                _logger.Info($"[TRANSFORMER TRACE] Cache Miss for key '{cacheKey}'. Initiating remote profile dispatch query to API.");
                try
                {
                    if (_tokenAcquisition == null) return principal;

                    var apiScopes = _configuration.GetSection("DownstreamApi:Scopes").Get<string[]>() ?? Array.Empty<string>();
                    var accessToken = await _tokenAcquisition.GetAccessTokenForUserAsync(apiScopes, OpenIdConnectDefaults.AuthenticationScheme, principal);

                    var client = _httpClientFactory.CreateClient("IMTHttpClient");
                    client.DefaultRequestHeaders.Authorization = new AuthenticationHeaderValue("Bearer", accessToken);

                    client.DefaultRequestHeaders.Add("X-User-Email", principal.FindFirst("preferred_username")?.Value ?? "unknown@domain.com");
                    client.DefaultRequestHeaders.Add("X-User-DisplayName", principal.FindFirst("name")?.Value ?? "Unknown User");

                    // ==================== CRITICAL FIX: DIRECT HEADER INJECTION ====================
                    string rolesCacheKey = $"UI_AppRoles_{azureAdObjectId.ToLowerInvariant()}";
                    if (_cache.TryGetValue(rolesCacheKey, out string appRoles) && !string.IsNullOrEmpty(appRoles))
                    {
                        client.DefaultRequestHeaders.Add("X-Entra-App-Roles", appRoles);
                        _logger.Info($"[TRANSFORMER TRACE] Directly stamped roles header onto /api/me request: '{appRoles}'");
                    }

                    var response = await client.GetAsync("me");
                    if (response.IsSuccessStatusCode)
                    {
                        var content = await response.Content.ReadAsStringAsync();
                        profile = JsonSerializer.Deserialize<UserProfileDto>(content, new JsonSerializerOptions { PropertyNameCaseInsensitive = true });

                        if (profile != null)
                        {
                            // ==================== FIX 2: STALE CACHE LIFESPAN GUARD ====================
                            bool isFallback = profile.Roles != null && profile.Roles.Contains("REQUESTER") && profile.Roles.Count == 1;
                            var cacheOptions = new MemoryCacheEntryOptions();
                            
                            if (isFallback) {
                                cacheOptions.SetAbsoluteExpiration(TimeSpan.FromSeconds(5));
                                _logger.Warn($"[TRANSFORMER TRACE] Received fallback REQUESTER role profile. Setting short-lived 5-second defensive cache expiration.");
                            } else {
                                cacheOptions.SetAbsoluteExpiration(TimeSpan.FromMinutes(60));
                                _logger.Info($"[TRANSFORMER TRACE] Received verified role profile. Setting standard 60-minute cache validation lifespan.");
                            }

                            _cache.Set(cacheKey, profile, cacheOptions);
                        }
                    }
                }
                catch (Exception ex)
                {
                    _logger.Error(ex, $"[TRANSFORMER CRITICAL] Exception occurred during profile synchronization: {ex.Message}");
                }
            }

            var clone = principal.Clone();
            var newIdentity = (ClaimsIdentity)clone.Identity!;

            if (profile != null)
            {
                newIdentity.AddClaim(new Claim("userId", profile.UserId.ToString()));
                
                // Enforce Single-Active Role Priority Matrix Rules using inline extraction
                int GetRolePriority(string roleName) => roleName?.ToUpper() switch { "ADMIN" => 1, "APPROVER" => 2, "SITEMANAGER" => 3, "REQUESTER" => 4, "DEVELOPER" => 5, _ => 99 };
                var topActiveRole = profile.Roles?.OrderBy(GetRolePriority).FirstOrDefault();
                
                if (!string.IsNullOrEmpty(topActiveRole))
                {
                    newIdentity.AddClaim(new Claim(ClaimTypes.Role, topActiveRole));
                }

                foreach (var perm in profile.Permissions ?? Enumerable.Empty<string>())
                {
                    newIdentity.AddClaim(new Claim("permission", perm));
                }
            }

            newIdentity.AddClaim(new Claim("UiRbacTransformed", "true"));
            return clone;
        }
    }
}

5. Backend API Architecture (Resource Server)

The API Layer functions as a stateless resource server. It extracts identity headers from the front-end, runs them through authorization checks, and handles real-time data provisioning on the fly.

5.1 `EntraIdRbacClaimsTransformer.cs`

Processes incoming endpoint requests by splitting them into two automated pipelines: an initial user registration gate (Path A) or an automatic role elevation sync gate (Path B).

using System;
using System.Collections.Generic;
using System.Linq;
using System.Security.Claims;
using System.Threading.Tasks;
using Microsoft.AspNetCore.Authentication;
using Microsoft.AspNetCore.Http;
using Microsoft.Extensions.Logging;

namespace InterventionManagement.API.Infrastructure.Security
{
    public class EntraIdRbacClaimsTransformer : IClaimsTransformation
    {
        private readonly ILogger<EntraIdRbacClaimsTransformer> _logger;
        private readonly IHttpContextAccessor _httpContextAccessor;
        private readonly IAccountRepository _accountRepository;

        public EntraIdRbacClaimsTransformer(ILogger<EntraIdRbacClaimsTransformer> logger, IHttpContextAccessor httpContextAccessor, IAccountRepository accountRepository)
        {
            _logger = logger;
            _httpContextAccessor = httpContextAccessor;
            _accountRepository = accountRepository;
        }

        public async Task<ClaimsPrincipal> TransformAsync(ClaimsPrincipal principal)
        {
            if (principal?.Identity == null || !principal.Identity.IsAuthenticated) return principal;
            if (principal.HasClaim(c => c.Type == "DbRbacTransformed")) return principal;

            var oidClaim = principal.FindFirst("http://schemas.microsoft.com/identity/claims/objectidentifier") ?? principal.FindFirst("oid");
            if (oidClaim == null) return principal;

            string azureAdObjectId = oidClaim.Value;
            var httpContext = _httpContextAccessor.HttpContext;
            var rolesFromHeader = new List<string>();
            
            var groupsHeader = httpContext?.Request.Headers["X-Entra-App-Roles"].FirstOrDefault();
            if (!string.IsNullOrEmpty(groupsHeader))
            {
                foreach (var group in groupsHeader.Split(','))
                {
                    var roleToken = group.Split('-').Last().ToUpper();
                    if (!rolesFromHeader.Contains(roleToken)) rolesFromHeader.Add(roleToken);
                }
            }

            var dbUser = await _accountRepository.GetDbUserByAzureAdObjectIdAsync(azureAdObjectId);

            // ==================== PATH A: JIT INITIAL PROVISIONING GATE ====================
            // Isolated completely from volatile permissions metrics to prevent unique constraint crashes
            if (dbUser == null)
            {
                _logger.LogInformation($"[API JIT PROVISIONING] User OID '{azureAdObjectId}' not found in database. Executing registration transaction.");
                var email = principal.FindFirst(ClaimTypes.Email)?.Value ?? httpContext?.Request.Headers["X-User-Email"].FirstOrDefault() ?? "unknown@domain.com";
                var displayName = principal.FindFirst("name")?.Value ?? httpContext?.Request.Headers["X-User-DisplayName"].FirstOrDefault() ?? "Unknown User";

                if (!rolesFromHeader.Any()) rolesFromHeader.Add("REQUESTER");

                await _accountRepository.ProvisionEntraIdUserAsync(azureAdObjectId, email, displayName, rolesFromHeader);
                dbUser = await _accountRepository.GetDbUserByAzureAdObjectIdAsync(azureAdObjectId);
            }
            // ==================== PATH B: CONTINUOUS SELF-HEALING SYNC GATE ====================
            else if (dbUser != null && dbUser.ActiveRole == "REQUESTER" && rolesFromHeader.Any(r => r != "REQUESTER"))
            {
                _logger.LogInformation($"[API RBAC SYNC] Self-healing role upgrade triggered for '{dbUser.Email}'. Upgrading REQUESTER to actual groups: '{string.Join(", ", rolesFromHeader)}'");
                await _accountRepository.UpdateUserRolesFromEntraIdAsync(azureAdObjectId, rolesFromHeader);
                dbUser = await _accountRepository.GetDbUserByAzureAdObjectIdAsync(azureAdObjectId);
            }

            var permissionCodes = await _accountRepository.GetPermissionCodesByAzureAdObjectIdAsync(azureAdObjectId);
            
            var clone = principal.Clone();
            var newIdentity = (ClaimsIdentity)clone.Identity!;
            
            if (dbUser != null)
            {
                newIdentity.AddClaim(new Claim("userId", dbUser.UserId.ToString()));
                newIdentity.AddClaim(new Claim(ClaimTypes.Role, dbUser.ActiveRole ?? "REQUESTER"));
            }
            foreach (var code in permissionCodes ?? Enumerable.Empty<string>())
            {
                newIdentity.AddClaim(new Claim("permission", code));
            }

            newIdentity.AddClaim(new Claim("DbRbacTransformed", "true"));
            return clone;
        }
    }
}

5.2 `AccountRepository.cs` (Dapper System Queries)

Encapsulates standard database interaction logic using explicit Dapper parameters wrapped inside atomic transaction blocks.

using System;
using System.Collections.Generic;
using System.Data;
using System.Linq;
using System.Threading.Tasks;
using Dapper;
using NLog;

namespace InterventionManagement.API.Infrastructure.Repositories
{
    public class AccountRepository : IAccountRepository
    {
        private readonly IDbContext _dbContext;
        private static readonly Logger _logger = LogManager.GetCurrentClassLogger();

        public AccountRepository(IDbContext dbContext)
        {
            _dbContext = dbContext;
        }

        public async Task ProvisionEntraIdUserAsync(string azureAdObjectId, string email, string displayName, List<string> roleNames)
        {
            using var connection = _dbContext.GetSqlConnection();
            connection.Open();
            using var transaction = connection.BeginTransaction();
            try
            {
                string insertUserSql = @"
                    INSERT INTO [IMT].[UM_Users] (AzureAdObjectId, Username, Email, DisplayName, IsActive, CreatedAt)
                    VALUES (@AzureAdObjectId, @Username, @Email, @DisplayName, 1, GETUTCDATE());
                    SELECT CAST(SCOPE_IDENTITY() as int);";

                var userParams = new DynamicParameters();
                userParams.Add("@AzureAdObjectId", azureAdObjectId, DbType.String);
                userParams.Add("@Username", email.Split('@')[0], DbType.String);
                userParams.Add("@Email", email, DbType.String);
                userParams.Add("@DisplayName", displayName, DbType.String);

                int newUserId = await connection.QuerySingleAsync<int>(insertUserSql, userParams, transaction);

                if (roleNames != null && roleNames.Any())
                {
                    string insertRolesSql = @"
                        INSERT INTO [IMT].[UM_UserRoles] (UserId, RoleId, AssignedAt)
                        SELECT @UserId, RoleId, GETUTCDATE()
                        FROM [IMT].[UM_Roles]
                        WHERE UPPER(RoleName) = UPPER(@RoleName);";

                    var roleParams = roleNames.Select(r => new { UserId = newUserId, RoleName = r });
                    await connection.ExecuteAsync(insertRolesSql, roleParams, transaction);
                }
                transaction.Commit();
            }
            catch (Exception ex)
            {
                transaction.Rollback();
                _logger.Error(ex, "Failed to provision user and roles transactionally.");
                throw;
            }
        }

        public async Task UpdateUserRolesFromEntraIdAsync(string azureAdObjectId, List<string> roleNames)
        {
            using var connection = _dbContext.GetSqlConnection();
            connection.Open();
            using var transaction = connection.BeginTransaction();
            try
            {
                string getUserIdSql = "SELECT UserId FROM [IMT].[UM_Users] WHERE AzureAdObjectId = @AzureAdObjectId;";
                int? userId = await connection.QueryFirstOrDefaultAsync<int?>(getUserIdSql, new { AzureAdObjectId = azureAdObjectId }, transaction);

                if (userId.HasValue)
                {
                    string deleteRolesSql = "DELETE FROM [IMT].[UM_UserRoles] WHERE UserId = @UserId;";
                    await connection.ExecuteAsync(deleteRolesSql, new { UserId = userId.Value }, transaction);

                    if (roleNames != null && roleNames.Any())
                    {
                        string insertRolesSql = @"
                            INSERT INTO [IMT].[UM_UserRoles] (UserId, RoleId, AssignedAt)
                            SELECT @UserId, RoleId, GETUTCDATE()
                            FROM [IMT].[UM_Roles]
                            WHERE UPPER(RoleName) = UPPER(@RoleName);";

                        var roleParams = roleNames.Select(r => new { UserId = userId.Value, RoleName = r });
                        await connection.ExecuteAsync(insertRolesSql, roleParams, transaction);
                    }
                }
                transaction.Commit();
            }
            catch (Exception ex)
            {
                transaction.Rollback();
                _logger.Error(ex, "Failed to update user roles transactionally.");
                throw;
            }
        }
    }
}

6. Cloud Infrastructure, Key Vault & Slot Configuration

The staging and production pipelines use infrastructure-level configuration bindings to keep secrets completely decoupled from source code control.

6.1 Azure Key Vault & Managed Identity Linkages

Sensitive settings, including the AzureAd:ClientSecret token and backend database strings, are isolated inside an Azure Key Vault container. At startup, the application logs into the vault using an system-assigned **Azure Managed Identity token**, pulling configurations directly into system RAM without writing secrets to local disk storage.

6.2 The Nested Key Double-Underscore Mapping Rule

To safely pass configurations down to Linux container instances, standard appsettings JSON colons (:) are automatically converted into **double underscores (__)** inside the Azure App Service configuration blade:

  • Local Machine JSON Configuration Target: AuthenticationFeatures:EnableSSO
  • Azure Portal Web App Configuration Key Name: AuthenticationFeatures__EnableSSO

6.3 Multi-Environment Sticky Slots Topology

To prevent configuration variables from getting scrambled during production rollouts, every environment setting is securely locked down using **Deployment Slot Settings (Sticky flags)**.

7. Branching Strategy & Zero-Downtime Rollout

The platform follows a strict lifecycle management process to ensure seamless integration and zero-downtime updates across water utility operation centers.

7.1 Branch Isolation Topology

All SSO-related refactoring was isolated to dedicated feature branches to keep the primary pipeline stable and uninterrupted during development:

  • Frontend MVC UI Subsystem Isolation Branch: feature_hby_SSO_12JUN26
  • Stateless Backend Resource Subsystem Isolation Branch: feature_hby_SSO_12JUN26

The code followed a progressive verification path: feature modifications were built and verified directly inside local environments first, then pushed to the Dev slot, and finally merged down to the main dev integration tracking branch before moving to the uat release target.

7.2 Zero-Downtime Swap Execution Routine

To perform seamless updates without dropping active database connections or user sessions, the platform uses an automated Blue-Green deployment slot workflow:

  1. The new codebase is deployed to the inactive **Staging Slot**, while the active production container continues handling real-time user traffic.
  2. Automated warmth diagnostics fire inside the staging environment to verify Key Vault synchronization links and database connectivity.
  3. Azure triggers an underlying network routing switch. The staging slot is promoted to production instantaneously. Because our environment variables are marked as **Sticky**, the containers swap seamlessly while remaining bound to their correct regional database configurations.

ARCHITECTURAL SAFEGUARD NOTE: If error spikes or exception logs are detected right after a swap, the routing engine automatically reverses the network switch. This brings the old container profile back online immediately, preventing any service disruptions for active users.
